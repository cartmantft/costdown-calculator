# 구현 퀵스타트 (Phase 1)

1) 브랜드 노출 확인  
- 브릿지 뷰/내비게이션 바에서 아이콘·이름·브랜드 색 일관 노출 확인.  
- 에셋 실패 시 이름만 보이는 대체 UI가 나오고 다음 진입에서 재시도하도록 구현.
- 검증: 토스 진입 시 1초 내 아이콘+이름+색 표시, 화면 전환 시에도 동일 아이콘 유지, 로고 로드 실패 시 이름-only 표시 후 재시도.

2) 줌 차단 적용  
- `index.html` 또는 루트 레이아웃에 `meta viewport`에 `initial-scale=1, maximum-scale=1, user-scalable=no` 추가.  
- 필요 시 멀티터치 줌 이벤트 차단(예: 터치 핸들러)으로 보강하되 탭/스크롤/입력은 영향 없게 유지.
- 검증: 주요 화면에서 핀치/멀티터치/더블탭 시 스케일 변화 없음, 탭·스크롤·입력 반응 정상.

3) 삭제 영속성 보장  
- 히스토리 로딩/저장 모두 로컬 스토리지 단일 경로로 일원화.  
- 삭제 시: 리스트 상태 업데이트 → 동일 데이터를 즉시 저장 → UI 리렌더.  
- 재실행 3회 이상 수동 검증: 삭제 항목 부활 0건 확인.
- 시나리오: 기록 추가 → 특정 항목 삭제 → 앱 재실행 3회 → 삭제 항목 재등장 0건, 나머지 순서 유지 → 모든 항목 삭제 시 빈 상태 유지 확인.

4) 한계값 및 검증  
- 히스토리 최대 10개, 분할 lot 최대 5개 유지.  
- 스키마 버전 필드로 마이그레이션 대비(기본 1).  
- 통화 기호/코드는 환경 설정과 일치하는지 확인.

5) 수동 체크리스트  
- 첫 화면(브릿지 뷰) 1초 내 이름·아이콘·색 노출.  
- 핀치/더블탭 줌 모두 무효.  
- 삭제 후 재실행 3회에서도 항목 재등장 없음.

6) 변경 기록  
- 2025-12-02: 브랜드 헤더/내비 추가, 줌 차단 메타/터치 가드 적용, 삭제 영속 로직 강화 및 수동 검증 시나리오 업데이트.
